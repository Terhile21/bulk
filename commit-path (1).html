Commit Path

The Commit Path is the bridge between BULK's high-speed off-chain world and Solana's blockchain. It takes all the trades happening in memory on the BULK Tile, proves they happened correctly using cryptographic signatures from validators, and writes that proof to Solana for permanent, trustless settlement.

How Settlement Works

Step 1: Collect State Changes

The Commit Path reads the state deltas produced by the BULK Tile from BULK Db. These deltas represent what changed after trades executed—balance updates, position changes, order fills.

Step 2: Batch the Changes

Instead of submitting every single trade to Solana individually, the Commit Path batches hundreds or thousands of trades together into a single settlement batch.

Step 3: Gather Validator Signatures

The Commit Path coordinates with other validators to collect cryptographic signatures. Each validator signs the batch, attesting that they verified these trades happened correctly on their Tile.

Step 4: Aggregate Signatures

Using BLS (Boneh-Lynn-Shacham) threshold signatures, all those individual validator signatures get combined into a single, compact signature.

Step 5: Submit to Solana

The batched state changes plus the aggregated signature get submitted to Solana as a single transaction. Solana verifies the signature and updates the on-chain state accordingly.

Why BLS Signatures?

BLS signatures have a special property: they're aggregatable. If 100 validators each sign the same data, those 100 signatures can be mathematically combined into one signature that proves all 100 signed it.

The math checks out: Solana verifies that the aggregated signature is valid and represents enough validator stake to meet the quorum threshold.

The Key Insight

The Commit Path embodies BULK's core design philosophy: do the fast work off-chain, prove it with cryptography, and settle it efficiently on-chain. This approach gives you centralized exchange performance with decentralized exchange security.
