BULK

Glossary

BLS Signatures

Boneh-Lynn-Shacham signatures. A type of cryptographic signature that can be aggregated—multiple signatures can be combined into one compact signature that proves all signers agreed.

Bulk-agave

The validator client software for BULK. It's a fork of Jito-agave that includes all BULK components while maintaining compatibility with Jito's MEV revenue streams.

BULK Db

The persistent storage layer that runs on each validator. A key-value, log-structured database that stores all state changes and trade fills, with peer replication across the network.

BULK Net

The high-speed order propagation network. Uses erasure coding and UDP fan-out to deliver orders to all validators globally in under 20 milliseconds.

BULK Tile

The in-memory execution engine that runs on each validator. Contains the matching engine, risk kernels, and liquidation engine.

Commit Path

The settlement process that bridges BULK and Solana. Batches trade results, collects validator signatures, and submits proofs to Solana's blockchain periodically.

Deterministic Execution

When multiple computers run the same program with the same inputs and produce exactly the same outputs.

Erasure Coding

A technique for breaking data into pieces with redundancy, so the original data can be reconstructed even if some pieces are lost.

In-Memory Execution

Running computations in RAM rather than on disk or on the blockchain. This is much faster but requires separate storage to make the results permanent.

Jito-agave

The most popular Solana validator client, known for enabling MEV extraction through blockspace auctions.

MEV (Maximal Extractable Value)

Revenue that validators earn by ordering, including, or excluding transactions in the blocks they produce.

Quorum

The minimum number or percentage of validators required to establish consensus.

State Delta

A record of what changed after an operation. The changes to balances, positions, and orders after trades execute.

UDP (User Datagram Protocol)

A network protocol that sends data without establishing connections or confirming delivery. Faster than TCP but less reliable.
