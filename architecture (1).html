BULK

Architecture

BULK uses a modular architecture that separates fast, time-sensitive operations from secure, on-chain settlement. Each component has a specific job, and together they create a trading system that's both high-performance and trustless.

The Five Core Components

Bulk-agave

The validator client. A fork of Jito-agave that lets validators run BULK while keeping their MEV revenue from blockspace auctions.

BULK Tile

The in-memory execution engine running on each validator. Handles matching, risk management, and liquidations at millisecond speed.

BULK Net

The order propagation network. Uses erasure coding and UDP broadcasting to spread orders globally in under 20 milliseconds.

BULK Db

The persistent storage layer. A key-value database that logs all state changes and replicates across validators for redundancy.

Commit Path

The settlement bridge. Batches trade results, collects validator signatures, and commits proofs to Solana's blockchain.

The Key Insight

BULK doesn't try to make the blockchain faster. Instead, it recognizes that not everything needs to happen on-chain immediately. By splitting execution (off-chain, fast) from settlement (on-chain, secure), BULK gets the best of both worlds.
