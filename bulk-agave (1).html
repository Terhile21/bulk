Bulk-agave

Bulk-agave is the foundation of the entire BULK system. It's the validator client software that nodes run to participate in the network. Think of it as the operating system that powers BULK's trading infrastructure.

What is Bulk-agave?

Bulk-agave is a strategic fork of Jito-agave, which is the most popular Solana validator client. This isn't a ground-up rewrite—it's Jito-agave with BULK's trading components integrated on top.

This design choice is critical. Validators already run Jito-agave because it gives them access to blockspace auctions, which generate significant MEV (Maximal Extractable Value) revenue. By building BULK as a fork rather than a replacement, validators don't have to choose between MEV income and BULK.

Why Fork Jito-agave?

Economic compatibility: Validators can earn from both Jito's blockspace auctions and BULK's order flow. They don't sacrifice revenue to run BULK.

Network adoption: Since Jito-agave is the dominant validator client, building on it means validators can adopt BULK without switching their entire infrastructure.

Proven stability: Jito-agave is battle-tested software running on thousands of validators. BULK inherits this reliability instead of building from scratch.

The key point: Bulk-agave isn't asking validators to run something new. It's giving them an upgraded version of what they already run, with BULK's trading capabilities built in.

Validator Economics

Validators running Bulk-agave earn from multiple sources: Jito MEV revenue, BULK order flow fees, and standard Solana rewards. This multi-source revenue model makes running Bulk-agave economically attractive.
